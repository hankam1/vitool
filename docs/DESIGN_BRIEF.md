# viTool — Дизайн-бриф («тур» для Claude Design)

> Этот документ — полное техническое задание на визуальный дизайн.
> Отдай его дизайнеру (или вставь в Claude/Figma AI) — на его основе можно собрать
> целостный UI без дополнительных вопросов. Стиль: **минималистичный тёмный, в духе Apple**.

---

## 0. TL;DR для дизайнера

Спроектируй desktop-приложение **viTool** — пользователь вставляет ссылку на YouTube-видео и
мгновенно получает: **транскрипт (скрипт) видео**, **превью (обложку) с кнопками скачивания**
и **дату + время публикации** (плюс канал, длительность, просмотры).

Нужно: один главный экран + состояния (пусто / загрузка / результат / ошибка) + лист настроек.
Эстетика — тёмная, тихая, «дорогая»: глубокий почти-чёрный фон, мягкая глубина, один спокойный
акцент (системный синий Apple), много воздуха, скруглённые поверхности, аккуратная типографика.

Целевые референсы по ощущению: **Raycast, Linear, Arc, Apple Music (dark), macOS System Settings, CleanShot X**.

## 1. О продукте

| | |
|---|---|
| **Название** | viTool |
| **Тип** | Лёгкое кросс-платформенное desktop-приложение (Tauri 2 — Windows / macOS / Linux) |
| **Суть** | YouTube-ссылка → транскрипт + превью + дата/время публикации |
| **Пользователь** | Контент-мейкеры, исследователи, студенты, журналисты — кому нужен текст видео и обложка быстро |
| **Ключевая ценность** | Скорость и тишина: вставил ссылку → получил всё нужное, без рекламы и лишнего |
| **Платформа отображения** | Окно ~920×680 (min 720×560), кастомный безрамочный заголовок |

### Что приложение делает (функциональные блоки на экране)
1. **Поле ввода ссылки** — вставка URL, кнопка «Анализ».
2. **Карточка превью** — обложка 16:9 + меню «Скачать превью» (разные разрешения).
3. **Метаданные** — название, канал, **дата и время публикации**, длительность, просмотры, кнопка «Открыть на YouTube».
4. **Панель транскрипта** — список сегментов (таймкод + текст), поиск, копирование, экспорт в TXT / SRT / VTT.

## 2. Задача дизайнеру (что отдать на выходе)

1. **High-fidelity макеты** каждого экрана во всех состояниях (см. §7): Welcome, Loading, Result, Errors (≥3), Settings.
2. **Component sheet** — все компоненты из §8 со всеми состояниями (default / hover / active / focus / disabled / loading).
3. **Таблица дизайн-токенов** (цвет, типографика, отступы, радиусы, тени, motion) — желательно сразу как CSS-переменные (см. §6, формат готов к копированию).
4. **Иконки** — единый набор в стиле SF Symbols / линейные, stroke 1.5 (см. §10).
5. **Спецификация микро-взаимодействий и анимаций** (§9).
6. (Опц.) Светлая тема как зеркало токенов — но **тёмная первична**, на ней фокус.

Формат: Figma-файл предпочтительно; либо аннотированные PNG + token-таблица. Сетка 4px.

## 3. Целевое ощущение (Mood)

- **Тихо и дорого.** Никакого визуального шума. Контент — герой, хром — невидим.
- **Глубокий тёмный, не «угольно-серый дешёвый».** Почти чёрный с лёгким холодным подтоном, многослойность поверхностей за счёт чуть более светлых карточек, а не жирных границ.
- **Apple-материальность.** Полупрозрачный заголовок с blur (vibrancy), тонкие hairline-разделители (1px, ~8% белого), мягкие тени, скругления 10–14px.
- **Один акцент.** Системный синий Apple (`#0A84FF`) — только для действий и фокуса. Всё остальное — нейтральная серо-белая шкала.
- **Движение — сдержанное.** Короткие плавные переходы 120–280ms, лёгкий «spring» на нажатиях. Никаких резких или долгих анимаций.

Одной фразой: **«Если бы Apple сделала маленькую утилиту для YouTube-транскриптов»**.

---

## 4. Принципы дизайна

1. **Content-first.** Транскрипт и превью занимают максимум пространства; элементы управления — компактные и по краям.
2. **Прогрессивное раскрытие.** На старте — только поле ввода. Всё остальное появляется после анализа.
3. **Одно главное действие на экране.** Welcome → «Анализ». Result → чтение/копирование/скачивание.
4. **Состояния обязательны.** Каждый блок имеет loading / empty / error. Скелетоны вместо спиннеров где можно.
5. **Тишина по умолчанию, обратная связь по требованию.** Тосты — короткие, ненавязчивые, авто-исчезают.
6. **Клавиатура — первый класс.** Ctrl/⌘+V вставка, Enter — анализ, Ctrl/⌘+C копирование, ⌘F поиск, Esc закрыть.
7. **Глубина светом, а не линиями.** Слои различаются яркостью поверхности + мягкой тенью, границы — минимальны.

---

## 5. Сетка, размеры окна, layout

- **Окно:** по умолчанию `920 × 680`, минимум `720 × 560`. Кастомный безрамочный titlebar высотой `40px` (drag-region).
- **Базовая сетка:** 4px. Все отступы/размеры кратны 4.
- **Внешние поля контента:** `24px` по бокам, `20px` сверху/снизу (на узком окне — `16px`).
- **Контентная колонка:** результат центрируется, max-width контента `760px`; транскрипт может тянуться на всю ширину колонки.
- **Карточка превью:** ширина ~`320px` (16:9 → высота ~180px), слева; метаданные — справа, гибко.
- **На узком окне (<760px):** превью и метаданные складываются вертикально (превью сверху).

> тонко докрутить, сохранив контраст и иерархию. Значения уже подобраны под WCAG AA для текста.

```css
  --bg-surface:   #141416;  /* карточки */

  --bg-hover:     #1F1F23;  /* ховер строк/кнопок */
  --bg-active:    #29292E;  /* нажатие */

  /* ─── Текст ─── */
  --text-primary:   #F5F5F7;  /* off-white (заголовки, основной) */
  --text-secondary: #A1A1A6;  /* подписи, метаданные */
  --text-tertiary:  #6E6E73;  /* подсказки, плейсхолдеры */
  --text-disabled:  #48484A;

  /* ─── Акцент (системный синий Apple, dark) ─── */
  --accent:         #0A84FF;
  --accent-hover:   #409CFF;
  --accent-active:  #0060DF;
  --accent-subtle:  rgba(10,132,255,0.14);  /* фон-подсветка акцента */
  --accent-text:    #FFFFFF;                /* текст на акценте */

  /* ─── Семантика (Apple dark system colors) ─── */
  --success: #30D158;
  --warning: #FFD60A;
  --error:   #FF453A;
  --info:    #64D2FF;
  --error-subtle:   rgba(255,69,58,0.14);
  --success-subtle: rgba(48,209,88,0.14);

  /* ─── Границы / разделители (hairlines) ─── */
  --border-subtle: rgba(255,255,255,0.07);
  --border:        rgba(255,255,255,0.10);
  --border-strong: rgba(255,255,255,0.16);
  --focus-ring:    rgba(10,132,255,0.55);

  /* ─── Радиусы ─── */
  --r-xs: 6px;
  --r-sm: 8px;
  --r-md: 10px;
  --r-lg: 14px;
  --r-xl: 20px;
  --r-pill: 999px;

  /* ─── Тени / высота ─── */
  --shadow-sm: 0 1px 2px rgba(0,0,0,0.40);
  --shadow-md: 0 4px 16px rgba(0,0,0,0.45);
  --shadow-lg: 0 12px 40px rgba(0,0,0,0.55);
  --highlight-top: inset 0 1px 0 rgba(255,255,255,0.05); /* верхний блик на поднятых поверхностях */

  /* ─── Размытие (vibrancy / материал) ─── */
  --blur-material: blur(20px) saturate(180%);

  /* ─── Типографика ─── */
  --font-ui:   "SF Pro Display", "SF Pro Text", -apple-system, "Inter", "Segoe UI", system-ui, sans-serif;
  --font-mono: "SF Mono", ui-monospace, "JetBrains Mono", "Cascadia Code", monospace;

  /* ─── Движение ─── */

  --dur-base: 180ms;
  --dur-slow: 280ms;

  --ease-out:      cubic-bezier(0.0, 0.0, 0.2, 1);

  /* ─── Spacing scale (4px base) ─── */
  --s-1: 4px;  --s-2: 8px;  --s-3: 12px; --s-4: 16px;
  --s-5: 20px; --s-6: 24px; --s-7: 32px; --s-8: 40px;
  --s-9: 48px; --s-10: 64px;

```

### Типографическая шкала

| Роль | Размер / интерлиньяж | Вес | Tracking | Применение |
|---|---|---|---|---|
| `display` | 28 / 34 | 600 | −0.02em | Заголовок welcome-экрана |
| `title-1` | 22 / 28 | 600 | −0.01em | Название видео |
| `title-2` | 17 / 22 | 600 | 0 | Заголовки секций («Транскрипт») |
| `headline` | 15 / 20 | 600 | 0 | Кнопки, акцентные подписи |
| `body` | 14 / 21 | 400 | 0 | Текст транскрипта, основной |
| `callout` | 13 / 18 | 400/500 | 0 | Метаданные, вторичные подписи |
| `caption` | 12 / 16 | 400 | 0 | Подсказки, хинты, статусы |
| `mono-time` | 12 / 16 | 500 | 0 | Таймкоды в транскрипте (`--font-mono`) |

> Шрифт: на macOS — системный SF Pro; на Windows/Linux — **поставляется Inter** (variable),
> чтобы вид был одинаковым. Таймкоды — моноширинным, чтобы колонка не «дрожала».

### S3 — Ошибки (минимум три варианта; единый паттерн «иконка + заголовок + пояснение + действие»)
1. **Неверная ссылка** — инлайн под полем ввода: красная hairline + текст «Не похоже на ссылку YouTube». Поле — `--error` border.
2. **Транскрипт недоступен** — карточка превью и метаданные показаны, а вместо списка транскрипта:
   `◌ У этого видео нет субтитров` + подпись «Автор не добавил субтитры, и автоматических тоже нет». (Превью/дата всё равно показаны!)
3. **Сеть/возрастное ограничение/приватное** — крупное пустое состояние по центру: иконка, «Не удалось получить данные», пояснение, кнопка «Повторить».

## 8. Компоненты (спецификация)

Для каждого — все состояния: `default / hover / active / focus-visible / disabled / loading`.

### 8.1 App chrome (кастомный titlebar)
- Высота `40px`, фон `--bg-base` с `--blur-material`, нижняя `--border-subtle`.
- Слева — место под системные кнопки окна (на macOS — нативные «светофоры»; на Windows/Linux — кастомные min/max/close справа в стиле Apple, маленькие круглые/плоские).
- По центру/слева — wordmark «viTool» (`headline`, `--text-secondary`); при результате — добавляется `· «название»` (усечение с …).
- Справа — иконка ⚙︎ Settings (icon-button).
- Вся пустая зона — `data-tauri-drag-region` (перетаскивание окна).

### 8.2 URL Input
- Высота `48px` (welcome) / `40px` (sticky). Фон `--bg-inset`, `--border` 1px, радиус `--r-md`.
- Внутри: иконка 🔗 (`--text-tertiary`), плейсхолдер `--text-tertiary`, кнопка-«вставить»/«очистить» справа.
- Кнопка анализа: либо встроенная ⏎ (welcome), либо `[Анализ]` (primary, sticky).
- **Focus:** граница `--accent`, кольцо `0 0 0 3px var(--focus-ring)`.
- **Error:** граница `--error`, под полем — `caption` красным.
- **Loading:** кнопка показывает спиннер, поле disabled.
- Поведение: автофокус на старте; валидация YouTube-URL (watch / youtu.be / shorts / embed); кнопка-«вставить» появляется, если поле пустое и в буфере есть ссылка.

### 8.3 Кнопки
- **Primary:** фон `--accent`, текст `--accent-text`, `headline`, высота `36px`, радиус `--r-md`, паддинг `0 16px`. Hover → `--accent-hover`; active → `--accent-active` + лёгкий `scale(0.98)` (spring); focus-ring.
- **Secondary:** фон `--bg-surface`, текст `--text-primary`, `--border`. Hover → `--bg-hover`.
- **Ghost / tertiary:** прозрачный, текст `--text-secondary`; hover → `--bg-hover` + текст `--text-primary`.
- **Icon-button:** `32×32`, радиус `--r-sm`, иконка `--text-secondary`; hover → `--bg-hover`.
- **Split / dropdown button** («Скачать ▾»): primary + отдельная зона со стрелкой, открывает поповер.
- Disabled: `opacity .4`, `cursor: not-allowed`, без hover.

### 8.4 Preview Card
- Контейнер `--bg-surface`, радиус `--r-lg`, `--shadow-md`, `--highlight-top`, паддинг `0` (обложка вплотную) либо `8px`.
- Обложка 16:9, радиус сверху `--r-lg`, `object-fit: cover`, плавный fade-in при загрузке (skeleton под ней).
- Поверх обложки снизу — лёгкий градиент-затемнение + кнопка «⬇ Скачать ▾» (или под обложкой).
- При наведении на обложку — очень лёгкое `scale(1.01)` + усиление тени.

### 8.5 Metadata block
- Вертикальный список строк «иконка + значение» (`callout`):
  - Канал (⦿ цветная точка-аватар или favicon), название канала.
  - 🗓 **Дата, время** — самый заметный из мета (можно `--text-primary`).
  - ⏱ Длительность · 👁 Просмотры (`--text-secondary`).
- Кнопка «Открыть на YouTube ↗» — ghost.
- Каждое значение можно скопировать по клику (микро-тост «Скопировано»).

### 8.6 Transcript panel
- **Header:** заголовок «Транскрипт» (`title-2`) слева; справа в ряд — поле поиска 🔎 (раскрывается по ⌘F), сегментированный контрол формата `TXT · SRT · VTT`, кнопка «⧉ Копировать всё».
- **Список сегментов:** область `--bg-inset`, радиус `--r-lg`, внутренний скролл, **виртуализация** для длинных видео.
- **Segment row:** колонка таймкода (`mono-time`, `--text-tertiary`, фикс. ширина ~56px) + текст (`body`, `--text-primary`). Паддинг `8px 12px`. Hover → фон `--bg-hover` + таймкод становится `--accent`. Клик → копировать сегмент (или выделить).
- **Поиск:** подсветка совпадений (`--accent-subtle` фон + `--accent` текст), счётчик «3/27», навигация ↑/↓ / Enter, авто-скролл к совпадению.
- **Язык:** если доступно несколько дорожек субтитров — компактный селектор языка в header (флаг/код), пометка «авто» для ASR.
- **Пусто:** «Нет субтитров» (см. S3-2).

### 8.7 Сегментированный контрол (TXT · SRT · VTT)
- Pill-контейнер `--bg-inset`, активный сегмент — «подушечка» `--bg-elevated` со `--shadow-sm`, плавный сдвиг (slide) при переключении. Текст активного — `--text-primary`, неактивного — `--text-secondary`.

### 8.8 Toast / уведомление
- Появляется снизу по центру, `--bg-elevated`, `--border`, `--shadow-lg`, радиус `--r-md`, паддинг `10px 14px`.
- Иконка статуса (✓ success / ⚠ error) + текст (`callout`). Авто-исчезает за 2.2s, slide-up + fade. Стопка при нескольких.

### 8.9 Skeleton (shimmer)
- Прямоугольники `--bg-surface` с бегущим бликом `--bg-hover` (animation 1.2s linear infinite, по `--ease-standard`). Радиусы как у реальных элементов.

### 8.10 Прочее
- **Tooltip:** `--bg-elevated`, `caption`, задержка 400ms.
- **Context-menu / Popover:** `--bg-elevated`, `--border`, `--shadow-lg`, радиус `--r-md`, пункты с hover `--bg-hover`, разделители `--border-subtle`.
- **Settings sheet:** затемнение фона (`rgba(0,0,0,.5)` + blur), панель `--bg-elevated` справа/по центру, радиус `--r-xl`.
- **Spinner:** тонкое кольцо (1.5px), `--accent`, для кнопок/инлайна. Полноэкранно — почти не используем (предпочитаем skeleton).
- **Focus-visible:** всегда видимое кольцо `0 0 0 3px var(--focus-ring)` на интерактивных элементах при навигации с клавиатуры.

## 9. Микро-взаимодействия и анимация

| Событие | Анимация | Длит. / easing |
|---|---|---|
| Появление результата | Карточка/транскрипт fade+slide-up 8px, со stagger 40ms | `--dur-slow` / `--ease-out` |
| Welcome → compact input | Поле «уезжает» наверх и сжимается | `--dur-slow` / `--ease-standard` |
| Нажатие кнопки | `scale(0.98)` | `--dur-fast` / `--ease-spring` |
| Hover строки/кнопки | Смена фона | `--dur-fast` / `--ease-standard` |
| Переключение TXT/SRT/VTT | Slide «подушечки» | `--dur-base` / `--ease-standard` |
| Toast | slide-up + fade in/out | `--dur-base` |
| Skeleton shimmer | Бегущий блик | 1.2s loop |
| Thumbnail load | Fade-in из скелетона | `--dur-slow` |
| Открытие поповера/меню | Scale 0.96→1 + fade, origin сверху | `--dur-fast` / `--ease-out` |

> Уважать `prefers-reduced-motion`: при включённом — оставлять только fade, без сдвигов/scale/shimmer.

## 10. Иконки

- Стиль: линейные, в духе **SF Symbols**, stroke **1.5px**, скруглённые концы, размер по умолчанию `18px` (в кнопках `16px`).
- Набор: 🔗 link, ⏎ enter, ⬇ download, ▾ chevron-down, ↗ external-link, 🔎 search, ⧉/copy, 🗓 calendar, ⏱ clock/duration, 👁 eye/views, ⚙︎ settings/gear, ✕ close, ⚠ warning, ✓ check, ◌ empty-doc, ⦿ channel-dot.
- Цвет по умолчанию `--text-secondary`; активные/акцентные — `--accent`.
- Рекомендованный источник: **Lucide** или **SF Symbols** (если только macOS). Единый набор, не смешивать стили.

## 11. Доступность (a11y)

- Контраст текста ≥ **4.5:1** (основной) и ≥ 3:1 (крупный) — палитра §6 это обеспечивает.
- Видимый **focus-ring** на всех интерактивных элементах (клавиатурная навигация).
- Полная клавиатурная управляемость (Tab-порядок логичен; шорткаты §4).
- `prefers-reduced-motion` — упрощённые анимации.
- Цель клика ≥ 32px. Иконки-кнопки — с `aria-label`.
- Не передавать смысл только цветом (иконка + текст в статусах/ошибках).

---

## 12. Адаптивность окна

- **Широкое (≥760px контент):** превью слева, метаданные справа, транскрипт во всю ширину под ними.
- **Узкое (<760px):** превью сверху на всю ширину, метаданные под ним, затем транскрипт. Sticky-поле и заголовок не ломаются.
- Минимальное окно `720×560` — всё должно оставаться usable.

## 13. Референсы / вдохновение (по ощущению)

- **Raycast** — компактность, поповеры, тёмная глубина, скорость.
- **Linear** — типографика, hairlines, спокойная палитра, аккуратные состояния.
- **Arc Browser** — мягкие поверхности, материал, лёгкая «игривость» в движении.
- **Apple Music / Podcasts (dark)** — карточки обложек, иерархия мета.
- **macOS System Settings (Ventura+)** — листы настроек, селекторы, тогглы.
- **CleanShot X / Cron (Notion Calendar)** — «дорогая» тишина и детали.

Чего НЕ хотим: дашборды, грид-карточки-плитки, неоновые градиенты, glassmorphism-перебор, тяжёлые тени, цветной фон, много акцентных цветов, иллюстрации-маскоты.

## 14. Анти-паттерны (избегать)

- ❌ Чисто чёрный `#000` фон большими плоскостями (выглядит «дёшево» на OLED-кромках) — берём `--bg-base #0C0C0E`.
- ❌ Жирные границы вместо разницы поверхностей.
- ❌ Несколько акцентных цветов одновременно.
- ❌ Спиннер на весь экран вместо скелетонов.
- ❌ Длинные (>300ms) или «пружинящие» переходы на крупных блоках.
- ❌ Плотная посадка (мелкие отступы) — нужен воздух.
- ❌ Модалки на каждое действие — предпочитаем инлайн и поповеры.

---

## 15. Микрокопирайт (tone of voice)

- Коротко, по-человечески, спокойно. Без восклицаний и канцелярита.
- Примеры:
  - Плейсхолдер: «Вставьте ссылку на YouTube-видео»
  - Кнопка: «Анализировать» / в работе «Анализ…»
  - Пусто-транскрипт: «У этого видео нет субтитров»
  - Ошибка сети: «Не удалось получить данные. Повторить?»
  - Тост: «Скопировано» / «Превью сохранено»
  - Подсказка про время: «Без ключа API показывается дата. С ключом — точное время.»
